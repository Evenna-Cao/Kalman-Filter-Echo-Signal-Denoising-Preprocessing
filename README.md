# Kalman Filter-based Radar Echo Signal Denoising and Preprocessing

## 📖 Project Overview
This project implements a robust **Kalman Filter Echo Signal Denoising Preprocessing** pipeline for radar systems. By integrating target kinematic characteristics (Constant Velocity Model) with radar observational data, it effectively fuses system dynamic models (predictions) with actual observations (measurements). This dynamic weight-balancing mechanism reliably filters out observation noise.

To isolate the target and eliminate background clutter, the system constructs a non-linear attenuation region centered on the predicted target position, applying a **2D Conical Suppression Window** in the Range-Doppler (RD) domain. This maximizes the suppression of background clutter while perfectly preserving the target signal. 

The output is a sequence of high Signal-to-Noise Ratio (SNR) RD images, laying a solid foundation for subsequent target recognition and classification tasks.

## 🏗️ System Architecture
The preprocessing module is built on a hierarchical 3-level architecture:
1. **Data Fusion Layer**: Associates track data and enables automatic matching. It handles heterogeneous data synchronization and establishes a timestamp alignment mechanism to ensure that raw echo frames perfectly correspond to trajectory data.
2. **Signal Processing Layer**: Utilizes Moving Target Detection (MTD) combined with Kalman filter state predictions to dynamically center the 2D conical clutter suppression window.
3. **Output Standardization Layer**: Converts the raw radar echoes and processed matrices into standard, high-fidelity RD image sequences (PNG) and structured data files (`.mat`).

## 📂 Code Structure & File Descriptions

### 1. `kaerman.m` (Interactive Visualization & Core Processing)
**Function**: The main interactive processing script. It provides a comprehensive graphical user interface (GUI) to visualize the target trajectory, the current radar observation, Kalman filter predictions, and the RD map before and after clutter suppression in real-time.
* **Key Features**:
    * Prompts the user to select the root directory containing `原始回波` (Raw Echoes), `点迹` (Point Tracks), and `航迹` (Tracks).
    * Initializes and continuously updates the 4D Kalman filter (Range, Azimuth, Range Velocity, Azimuth Velocity).
    * Applies a Taylor window and FFT for MTD processing.
    * Optionally exports the mathematical processing results to a `processedResults.mat` file for offline analysis.
* **Usage**: Run the script in MATLAB. Use the UI dialog to select the data folder, and use the on-screen "Pause", "Resume", and "Stop" buttons to control the frame-by-frame visualization.

### 2. `kaerman_readme.m` (Post-Processing Analysis Tutorial)
**Function**: A utility and documentation script. The top half contains a commented-out tutorial explaining how to load, parse, and analyze the `processedResults.mat` file generated by `kaerman.m`. 
* **Key Features**:
    * Demonstrates how to extract frame-specific data (Doppler axis, Range axis, suppression matrices).
    * Provides boilerplate code for plotting trajectory analyses, calculating Kalman prediction errors, and extracting target features (SNR, amplitude).
    * *Note: The bottom half of this file contains a backup of the batch-processing script logic.*
* **Usage**: Read the commented section for data structure definitions, or copy-paste the snippet blocks into a new script to perform statistical analysis on your processed `.mat` files.

### 3. `kaerman_shuchutupian.m` (Single-File Automated Image Extraction)
**Function**: A headless (no GUI) processing script tailored for processing a single, hardcoded radar data triplet (one `.dat` echo file, one `.txt` point file, and one `.txt` track file). 
* **Key Features**:
    * Processes the data seamlessly without opening interactive visualization windows.
    * Automatically plots the filtered RD maps and saves them directly as sequence images (e.g., `1.png`, `2.png`) into a designated output directory.
* **Usage**: Update the `rawDataFile`, `pointFile`, `trackFile`, and `outputDir` variables at the top of the script with your local paths. Run the script to generate frame-by-frame RD images.

### 4. `kaerman_picture.m` (Batch Processing with Kalman Filter)
**Function**: A robust batch-processing script designed to handle entire datasets. It automatically traverses a source folder, matches raw echo `.dat` files with their corresponding point and track `.txt` files using Regular Expressions (`serial1_Label_serial2`), and processes them.
* **Key Features**:
    * Uses the Kalman filter prediction to dynamically locate the target and apply the adaptive suppression window.
    * Automatically creates subdirectories based on the dataset labels (`serial2`) to keep output images organized.
* **Usage**: Set `folderA` to your dataset root and `folderB` to your desired output directory. Run the script to process all files in bulk.

### 5. `picture1.m` (Batch Processing Baseline - Direct Observation)
**Function**: Similar to `kaerman_picture.m` in its batch-processing capabilities and folder traversal, but with a critical mathematical difference: **it does not apply the Kalman Filter for prediction.**
* **Key Features**:
    * Centers the 2D conical suppression window based strictly on the *true* multi-Doppler and range data directly extracted from the Point Track data, rather than a predictive model.
    * Acts as a baseline or control group to evaluate the performance improvement brought by the Kalman Filter dynamic weighting.
* **Usage**: Set `folderA` and `folderB` paths and run. Use this script when you want to generate RD maps relying purely on raw point measurements without predictive smoothing.

## 🚀 Quick Start Guide

1. **Prerequisites**:
   * MATLAB (R2021a or newer recommended).
   * Signal Processing Toolbox (required for `taylorwin` and FFT functions).
2. **Data Preparation**: 
   Ensure your data is organized into three subfolders under a root directory (e.g., `Dataset/`):
   * `原始回波/` (Contains `*_Label_*.dat` files)
   * `点迹/` (Contains `PointTracks_*.txt` files)
   * `航迹/` (Contains `Tracks_*.txt` files)
3. **Execution**:
   * For visual inspection and algorithm tuning: Run `kaerman.m`.
   * For dataset generation (creating images for Neural Network training): Run `kaerman_picture.m`.

## ⚙️ Key Parameters Setup
If you need to adjust the radar system constants, locate the **Global Variables** section at the top of each script:
```matlab
Fs = 20e6;              % Sampling Rate (20 MHz)
delta_R = 3e8/2/Fs;     % Range Resolution
```
To adjust the aggressiveness of the background noise filtering, modify the `suppressionRadius` in the main loop:
```matlab
suppressionRadius = 5;  % Increase to widen the target preservation zone
```
